# Ralph Universal Template

The Universal Template system allows you to initialize and scaffold projects for any platform or technology stack using an interactive CLI.

## Overview

Ralph Universal Template provides:
- **Interactive CLI** (`init-ralph.sh`) - Asks questions to determine project requirements
- **Project Scaffolding** (`scaffold.sh`) - Generates project structure based on configuration
- **PRD Generation** (`generate-prd-from-config.sh`) - Creates PRD with initial user stories
- **Rule Selection** - Automatically selects relevant Cursor Rules for your project type

## Quick Start

### 1. Initialize a New Project

```bash
./init-ralph.sh
```

This interactive tool will ask you questions about:
- Project name and description
- Project type (website, web app, iOS, Android, desktop, CLI, API, full stack)
- Technology stack (frontend, backend, database)
- Features (authentication, payments, real-time, etc.)
- Deployment platform
- Testing approach

### 2. Scaffold the Project

```bash
./scaffold.sh
```

This creates the project directory structure, copies template files, and sets up:
- Project structure
- `.cursorrules` with appropriate rules
- Ralph workflow files
- Initial configuration

### 3. Generate PRD

```bash
./generate-prd-from-config.sh
```

This creates `prd.json` with initial user stories based on your project configuration.

### 4. Start Development

```bash
cd <project-name>
./ralph.sh
```

Begin the Ralph autonomous development workflow!

## Supported Project Types

### Websites
- **Template:** Next.js static site
- **Default Stack:** Next.js, TypeScript, Tailwind CSS
- **Best For:** Marketing sites, portfolios, blogs
- **Rules:** nextjs, seo, tailwind-css, vercel

### Web Apps
- **Templates:** Next.js, React SPA
- **Default Stack:** React/Next.js, TypeScript, Tailwind CSS
- **Best For:** Interactive web applications
- **Rules:** react, nextjs, typescript, backend-development

### iOS Apps
- **Template:** Swift/SwiftUI native app
- **Default Stack:** Swift, SwiftUI
- **Best For:** Native iOS applications
- **Rules:** ios-app, testing, security

### Android Apps
- **Template:** Kotlin/Jetpack Compose native app
- **Default Stack:** Kotlin, Jetpack Compose
- **Best For:** Native Android applications
- **Rules:** testing, security

### Desktop Apps
- **Template:** Electron cross-platform
- **Default Stack:** Electron, Node.js
- **Best For:** Cross-platform desktop applications
- **Rules:** security, performance

### CLI Tools
- **Template:** Node.js or Python CLI
- **Default Stack:** Node.js or Python
- **Best For:** Command-line utilities
- **Rules:** testing

### API Backend
- **Template:** FastAPI, Node.js, or Django
- **Default Stack:** FastAPI/Node.js, PostgreSQL
- **Best For:** REST/GraphQL APIs
- **Rules:** fastapi, backend-development, postgresql, testing

### Full Stack
- **Template:** Combined frontend + backend
- **Default Stack:** React/Next.js + FastAPI/Node.js + PostgreSQL
- **Best For:** Complete applications
- **Rules:** react, nextjs, backend-development, fastapi, postgresql

## Configuration Files

### project-config.json

Generated by `init-ralph.sh`, contains all project settings:

```json
{
  "projectName": "my-app",
  "projectType": "webapp",
  "platform": "web",
  "techStack": {
    "frontend": "nextjs",
    "backend": "fastapi",
    "database": "postgresql",
    "mobile": "none"
  },
  "features": ["auth", "payments"],
  "deployment": "vercel",
  "testing": ["unit", "e2e"],
  "description": "My awesome app",
  "branchName": "ralph/my-app"
}
```

### rule-selector.json

Maps project types and tech stacks to Cursor Rules. Used automatically during scaffolding.

## Templates

Templates are located in `templates/` directory. Each template includes:
- Basic project structure
- Configuration files (package.json, etc.)
- Starter code
- README.md

### Customizing Templates

1. Edit files in `templates/<template-name>/`
2. Use placeholders like `{{PROJECT_NAME}}` for variable substitution
3. Template files are copied during scaffolding

## Workflow

```
init-ralph.sh → project-config.json
                     ↓
              scaffold.sh → project structure
                     ↓
    generate-prd-from-config.sh → prd.json
                     ↓
                  ralph.sh → development
```

## Rule Selection

Ralph automatically selects relevant Cursor Rules based on:
- Project type
- Technology stack
- Selected features
- Deployment platform
- Testing approach

Rules are referenced in the generated `.cursorrules` file, which guides Cursor AI during development.

## Advanced Usage

### Manual Configuration

You can manually edit `project-config.json` instead of using the interactive CLI:

```bash
# Edit project-config.json
vim project-config.json

# Then scaffold
./scaffold.sh
```

### Custom Templates

Create custom templates in `templates/` directory:

```bash
mkdir templates/my-custom-template
# Add files to template
# Update scaffold.sh to handle your template
```

## Troubleshooting

### Template Not Found

If you get "Template not found" error:
1. Check that template exists in `templates/`
2. Verify template name matches project type in scaffold.sh
3. Create minimal template if missing

### PRD Generation Issues

If PRD generation fails:
1. Verify `project-config.json` is valid JSON
2. Check that jq is installed: `jq --version`
3. Verify required fields are present

### Rule Selection Not Working

If rules aren't being selected:
1. Check `rule-selector.json` has entry for your project type
2. Verify rule files exist in `.cursor/rules/`
3. Check generated `.cursorrules` file
